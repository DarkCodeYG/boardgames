## 1. 아그리콜라 2016 개정판 vs 구버전 핵심 차이

| 항목 | 원판 (2007) | 개정판 (2016) |
|------|-----------|-------------|
| 카드 수 | 직업 169 + 소시설 139 = 308장 | 직업 48 + 소시설 48 = **96장** |
| 인원 | 1-5인 | **1-4인** (5-6인 확장팩 별매) |
| 동물 토큰 | 일반 나무 큐브 | **Animeeples** (동물 모양 조각) |
| 보드 확장 | 카드 기반 | **퍼즐 조각형** 보드 확장 (인원별 다름) |
| 카드 덱 | Z덱(기본) + 다수 확장 | **E/I/K 3단계** 난이도 분류 |
| 카드 품질 | 약한 카드 포함 (나쁜 핸드 가능) | 모든 카드 검증 완료 (우열카드 제거) |
| 스코어보드 | 별도 참조 카드 | **메인 보드에 통합** |

### 구현 시 혼동 주의 사항
- BGG에는 원판(#31260)과 개정판(#200680)이 별도 등재됨
- 카드 코드(Z-, A-, E-, I-, K- 등)는 버전별로 다름 → 개정판 카드만 사용
- 온라인 룰 검색 시 구버전 룰(5인, 구카드 텍스트)이 혼재하므로 주의

## 3. 게임 구조 (규칙 요약)

### 전체 흐름
- **6스테이지 × 총 14라운드**
- 각 라운드: ①라운드 카드 공개 → ②자원 보충 → ③워커 배치 → ④회수

### 수확 (6회, 각 스테이지 끝)
1. 밭 단계: 씨 뿌린 칸에서 자원 1개 수확
2. 식량 단계: 가족 1인당 음식 2개 지불 (부족 시 구걸 토큰)
3. 번식 단계: 같은 종 동물 2마리 이상 보유 시 1마리 추가

### 행동 공간
- 영구 행동 공간: 14개 (인원별 일부 다름)
- 라운드 카드로 추가: 14개 (라운드마다 1개씩 공개, 이후 유지)

### 점수
- 방·밭·울타리·직업·소시설·가족·자원별 VP
- 구걸 토큰: -3점/개
- 가족 5명 완성 보너스

## 4. 기술 복잡도 분석

### 4-1. 고난이도 구현 항목

| 항목 | 난이도 | 이유 |
|------|--------|------|
| **농장 보드 시각화** | ★★★★★ | 3×5 격자에 울타리(선분) + 방(셀) + 밭(셀) + 외양간(셀) + 동물(목장 단위) 혼재 |
| **카드 효과 엔진** | ★★★★★ | 96장 × 고유 트리거/조건/효과, 타이밍 규칙, 상호작용 |
| **울타리/목장 계산** | ★★★★☆ | 울타리 배치 → 연결된 목장 자동 계산, 유효성 검사 |
| **멀티플레이어 동기화** | ★★★★☆ | 워커 배치 순서, 수확 동시 처리, Firebase 트랜잭션 |
| **동적 행동 공간** | ★★★☆☆ | 인원별 다른 보드 구성 + 라운드마다 추가 |
| **점수 계산** | ★★★☆☆ | 다차원 점수 (방 종류별, 자원별 등급제) |
| **번식 로직** | ★★☆☆☆ | 종별 최대 수용량, 목장 용량 제한 |
| **직업 발동 시점** | ★★★★☆ | 카드별 발동 타이밍 (배치 시, 수확 시, 특정 행동 후 등) |

### 4-2. 농장 보드 데이터 구조 (핵심 설계 결정)

```
FarmBoard = {
  cells: Cell[3][5]          // 3행 5열
  fences: FenceSet           // 세로 울타리 3×4 + 가로 울타리 4×5
  pastures: Pasture[]        // 울타리로 둘러싸인 연결 구역 (자동 계산)
}

Cell = {
  type: 'empty' | 'room_wood' | 'room_clay' | 'room_stone' | 'field' | 'stable'
  animals?: { type: AnimalType, count: number }
}

FenceSet = {
  horizontal: boolean[4][5]  // 셀 위쪽 선분
  vertical: boolean[3][4]    // 셀 오른쪽 선분
}
```

→ 목장 계산은 Flood-fill 알고리즘으로 연결 구역 자동 탐지

### 4-3. 카드 효과 엔진 설계 요구사항

```
CardEffect = {
  trigger: TriggerType        // PLACE_WORKER | HARVEST | BREEDING | BUILD | etc.
  condition?: Condition       // 조건 (자원 X 보유 시, 특정 행동 시 등)
  effect: EffectFn           // 순수 함수: (state, player) => newState
  timing: 'before' | 'after' | 'instead'
}
```

## 5. 가능성 결론

### 기술 스택 적합성
현재 boardgames 프로젝트 스택 (React 19 + TypeScript + Zustand + Firebase)은 아그리콜라 구현에 **적합**:
- Firebase Realtime DB: 실시간 워커 배치/수확 동기화 가능
- Zustand: 복잡한 게임 상태 관리 가능
- React: 농장 보드 격자 렌더링 가능 (CSS Grid)
- TypeScript: 카드/상태 타입 안전성 확보

### 레퍼런스 구현 존재
- Asmodee Digital Steam 버전 (공식) — 구현 가능성 검증됨
- Board Game Arena 웹 버전 — 웹 구현 가능성 검증됨
- GitHub e2crawfo/agricola — AI 학습용 Python 구현 (데이터 참조 가능)

### 개발 규모 예측
| 단계 | 범위 | 예상 시간 |
|------|------|-----------|
| Phase 1 | 게임 엔진 + 농장 보드 + 행동 공간 | 대형 |
| Phase 2 | 카드 데이터베이스 + 효과 엔진 | 대형 |
| Phase 3 | 멀티플레이어 + Firebase | 중형 |
| Phase 4 | UI 완성 + 테스트 | 중형 |
| Phase 5 | 한/영/중 i18n + 튜토리얼 | 소형 |

### 최종 판정: **구현 가능, 단계적 접근 필수**

> 1라운드 진행 가능한 최소 MVP → 14라운드 완전 게임 → 카드 효과 완성 → 멀티플레이어 순서로 진행 권장
